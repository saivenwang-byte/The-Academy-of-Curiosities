## 4. 部署决策：跳过

| 原因 | 说明 |
|------|------|
| **场景不匹配** | 图书印前 vs 自媒体配图 |
| **无开源 CLI 图书管线** | Skill 依赖 Codex 会话 + 图像 API，非可复现 `pip install` 排版器 |
| **版权与风格** | 泛用 AI 手绘风与 [`00_插画师视觉创作说明书_V1.0.md`](../../05_视觉设定/00_插画师视觉创作说明书_V1.0.md) 原创 IP 要求冲突 |
| **已有路径** | Reader HTML/CSS → PDF 过渡 · E22 InDesign 为终态 |
| **视频证据不完整** | 未能转写口播中的具体仓库 URL；即便部署也需人工确认单一 repo，收益低于 E22 发包 |

**未执行**：`git clone` · `npm install` · Skill 拷贝至 `tools/`。

## 5. 若未来需「示意配图」的替代建议

（非本次任务范围 · 仅供主编备忘）

1. **正典插图**：继续 `样章包/插图/depth_anchor/` 锚点包 + 画师精修；
2. **版式打样**：优先 E22 方向 B 32p **实印**；
3. **内审草稿**：可用项目已有 `generate_vol1_illustrations.py` 或画师 brief，**不**引入第三方「笔记配图」Skill 进入出版链。

## 6. 阻塞项

- 头条视频 **无法在自动化环境完整播放**（CDN/登录墙）；
- 口播中 **具体 GitHub 仓库名未从视频音轨核实**（标题仅泛指「Github 宝藏项目」）；
- 即便定位为边缘探索，也 **不满足**「可合法下载并部署进本 repo 图书排版管线」的采纳门槛。
