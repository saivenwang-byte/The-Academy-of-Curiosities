## 0. 一句话定义

| 概念 | 是什么 |
|------|--------|
| **`/loop`（Claude Code 官方）** | 用 cron 式调度让 Agent **定时重复**执行同一类任务（5m / 30m / 每天） |
| **Ralph Loop** | 让 Agent **在同一会话内反复迭代同一任务**，直到**机器可验证的完成条件**满足；靠 **Stop Hook** 拦截「想退出」 |
| **Ralph 哲学** | Geoffrey Huntley：**「Ralph is a Bash loop」** — 反复喂同一份 prompt；**文件系统 + git** 当跨轮记忆 |

头条文章核心判断（保留、已校正）：

- 编程范式从「手写 → 单次 Prompt → **Loop 编排**」迁移  
- 人的工作从写代码变为 **写 Loop、定完成标准、验结果**  
- Ralph Loop 是社区在 `/loop` 之上做的 **「直到真完成才停」** 增强，非替代官方 `/loop`

---

## 1. 三种范式对照（学习用）

```
手写代码          → 人执行每一步
单次 Prompt       → AI 做一轮，人再 prompt
/loop             → 定时重复（看护 CI、抓反馈、依赖 PR）
Ralph Loop        → 同任务多轮迭代，直到测试/标签说 DONE
Dynamic Workflows → 多 Agent 并行/流水线（Fan-out、对抗验证等）
```

| 维度 | 单次 Prompt | `/loop` | Ralph Loop |
|------|-------------|---------|------------|
| 触发 | 手动 | 时间表 | 完成条件未满足则自动下一轮 |
| 记忆 | 会话内 | 每 tick 新上下文 | **磁盘文件 + git** 持久 |
| 停止 | 模型自觉停 | 永不自动停（需人关） | `--max-iterations` + completion promise |
| 适合 | 小改、问答 | 监控、汇总 | 测试驱动、批量 refactor、夜间任务 |

## 2. 环境准备

### 2.1 Claude Code 路线（官方 Ralph 插件）

| 项 | 要求 |
|----|------|
| CLI | [Claude Code](https://docs.anthropic.com/en/docs/claude-code) 已安装并登录 |
| 插件市场 | 可访问 `claude-plugins-official` |
| **Windows** | **Git for Windows**（Stop Hook 依赖 `bash`） |
| 可选 | Docker（AFK/夜间跑时的沙箱） |

**Windows 常见坑**：系统 `bash` 指向 WSL 会导致 Hook 失败。官方 workaround：

编辑缓存插件内 `hooks/hooks.json`：

```json
"command": "\"C:/Program Files/Git/bin/bash.exe\" ${CLAUDE_PLUGIN_ROOT}/hooks/stop-hook.sh"
```

路径：`~/.claude/plugins/cache/claude-plugins-official/ralph-wiggum/*/hooks/hooks.json`  
（用 `Git/bin/bash.exe`，不要用 `Git/usr/bin/bash.exe`）

### 2.2 Cursor 路线（你当前 IDE）

| 能力 | 说明 |
|------|------|
| **Multitask / 后台 Agent** | 协调器派生子 Agent，类似「一次 Ralph 迭代」 |
| **`/loop` skill** | 本地定时唤醒 Agent（`C:\Users\Lenovo\.cursor\skills-cursor\loop\SKILL.md`）— **定时重复**，不是 Stop Hook 式 Ralph |
| **社区 Ralph** | [ralph-wiggum-cursor](https://github.com/agrimsingh/ralph-wiggum-cursor) 等：shell 循环 + `RALPH_TASK.md` |

**结论**：在 Cursor 上要完整 Ralph 语义，选 **社区 shell 循环** 或 **手搓 ralph.sh**；要「每 5 分钟看一眼 CI」用 **Cursor `/loop` skill** 或 Claude Code **`/loop`**。

## 3. 安装

### 3.1 方法一 · Claude Code 官方插件（推荐）

```bash
# 在 Claude Code 会话内
/plugin install ralph-loop@claude-plugins-official
```

验证：

```bash
/help
# 应能看到 /ralph-loop 、 /cancel-ralph
```

### 3.2 方法二 · 手动 Bash Ralph（任意 CLI）

项目根目录：

**`ralph.sh`**

```bash
#!/bin/bash
set -e

MAX_ITERATIONS=${1:-10}
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"

echo "🔄 Starting Ralph Loop (max=$MAX_ITERATIONS)"

for i in $(seq 1 $MAX_ITERATIONS); do
  echo "═══ Iteration $i / $MAX_ITERATIONS ═══"

  OUTPUT=$(cat "$SCRIPT_DIR/prompt.md" \
    | claude --continue 2>&1 \
    | tee /dev/stderr) || true

  if echo "$OUTPUT" | grep -q "<promise>COMPLETE</promise>"; then
    echo "✅ Done at iteration $i"
    exit 0
  fi

  sleep 2
done

echo "⚠️ Max iterations reached without COMPLETE"
exit 1
```

**`prompt.md`**（任务 SSOT · 每轮不变）

```markdown
# Ralph Agent Instructions

## Task
（写清要做什么）

## Completion Criteria（机器可验证）
- [ ] 所有测试通过：`npm test` 或 `pytest`
- [ ] 构建无错误
- [ ] （可选）覆盖率 > 80%

## Stop Condition
全部完成后，且仅此时，输出一行：
<promise>COMPLETE</promise>
否则继续修改代码并再跑测试。
```

```bash
chmod +x ralph.sh
./ralph.sh 25
```

**Windows PowerShell 等价思路**：`for ($i=1; $i -le 25; $i++) { Get-Content prompt.md | claude ... }`，完成条件仍 grep `COMPLETE`。

### 3.3 方法三 · Cursor 社区 Ralph

## 4. 学习路径（HITL → AFK）

头条与 [aihero.dev 11 Tips](https://www.aihero.dev/tips-for-ai-coding-with-ralph-wiggum) 共识：

| 阶段 | 模式 | 做法 |
|------|------|------|
| **L1 观察** | HITL | 只跑 **1 次**迭代，看 Agent 改什么、测什么、何时误判完成 |
| **L2 修 prompt** | HITL | 把 completion 改成 **可执行命令**（`npm test` 退出码 0） |
| **L3 小 Loop** | AFK | `--max-iterations 5` · 修 lint / 单文件测试 |
| **L4 生产 Loop** | AFK | 独立 **git worktree/分支** · Docker 沙箱 · 30–50 次 cap |
| **L5 编排** | Loop 体系 | 多个 `/loop` 看护 + 偶尔 Ralph 深任务 |

**练习任务（按难度）**

1. 「修到 `eslint` 零 error」— max 10  
2. 「为 `utils/` 补测试至 coverage>80%」— max 30  
3. 「依赖 minor 升级 + CI 绿」— max 20

## 5. 部署与运行（生产 checklist）

### 5.1 启动 Ralph（Claude Code 官方）

```bash
/ralph-loop "为当前项目添加单元测试。
完成标准:
- 所有测试通过
- 测试覆盖率 > 80%
- 输出 <promise>COMPLETE</promise>" \
  --completion-promise "COMPLETE" \
  --max-iterations 50
```

取消：

```bash
/cancel-ralph
```

### 5.2 启动基础 `/loop`（Claude Code · 非 Ralph）

```bash
/loop 5m check if the deployment finished and tell me what happened

/loop check whether CI passed and address any review comments

/loop 30m fetch the latest tweets mentioning our product and summarize them
```

### 5.3 Prompt 模板（直接复用）

```markdown
## Task
（单一目标，不要「整个电商平台」）

## Constraints
- 不改 xxx 目录
- 不 git push
- 每轮先读 progress.txt / git log

## Verification（每轮必跑）
1. `npm test`
2. `npm run build`

## Done
全部通过后输出：<promise>COMPLETE</promise>

## If stuck（escape hatch）
超过 15 轮仍失败则：
- 写 BLOCKERS.md
- 输出 <promise>BLOCKED</promise>（注意：官方 plugin 的 --completion-promise 只认一个字符串，BLOCKED 需另设 max-iterations 或改 prompt）
```

### 5.4 并行 Loop 体系（Boris 式 · 参考）

| Loop | 周期 | 职责 |
|------|------|------|
| PR 看护 | 事件/5m | CI 绿 · rebase · 简单 review 意见 |
| 依赖 | 1h | 安全 minor 升级 → 开 PR |
| 反馈 | 30m | 社媒/工单聚类 |
| 质量 | 每日 | 静态分析 · flaky test |
| 深任务 | 夜间 Ralph | 大 refactor / 补测试 |

## 6. 最佳实践（来自头条 + 官方 + 社区）

1. **完成标准必须机器可验证** — 测试退出码、grep 标签，避免「感觉好了」  
2. **永远设 `--max-iterations`** — completion promise 是精确字符串匹配，不能多态  
3. **小步迭代** — 每轮只啃一个小目标，便于回滚  
4. **独立分支 / worktree** — Ralph 会反复改文件  
5. **AFK 必 Docker 沙箱** — 限制 `rm -rf ~` 类事故  
6. **progress.txt 入库** — 每轮简短记录（Anthropic 长跑 Agent .harness 建议）  
7. **成本** — 简单子任务用 Sonnet；Ralph 50 轮 Opus 可能烧额度，先 HITL 估 token  
8. **不适合 Ralph** — 需审美/产品取舍的设计稿、生产 incident 精确定位（用 targeted debug）

## 7. 与 Dynamic Workflows 的关系（延伸阅读）

Anthropic **Dynamic Workflows**（2026-05 发布）解决：

- Agentic Laziness（做一半就停）  
- Self-preferential Bias（自己验自己）  
- Goal Drift（多轮丢约束）

API 概念：`agent()` · `parallel()` · `pipeline()`  
常用组合：**Fan-out → 对抗验证 → loop until done**

**与 Ralph 分工**：Workflow = **多角色编排**；Ralph = **单角色反复直到 DONE**。可组合：Workflow 里某一 stage 用 Ralph。

### 8.2 版本对照表

| 组件 | 查看版本 | 升级渠道 |
|------|----------|----------|

### 8.3 成本与上限调优

| 信号 | 动作 |
|------|------|
| 经常打满 max 未 COMPLETE | 拆任务 / 改 verification / 降 scope |
| 3 轮内就 COMPLETE 但质量差 | 加强测试 / 加 adversarial review 子 agent |
| 账单过高 | 降 max-iterations · 换 Sonnet · 仅夜间跑 Ralph |

## 9. 故障排查

| 现象 | 原因 | 处理 |
|------|------|------|
| Hook 不循环 | Windows bash 指向 WSL | §2.1 Git Bash 路径 |
| 无限空转 | 无测试 / 标准模糊 | 加 `npm test` 硬门槛 |
| 过早 COMPLETE | promise 太易输出 | prompt 禁止未测完就 promise |
| 改坏 main | 未隔离分支 | worktree + 仅 merge 通过 CI 的 PR |
| Cursor 无 `/ralph-loop` | 非 Claude Code | 用 §3.2 或 §3.3 |

## 10. 本仓库可落地的 Loop 示例（《学堂趣事录》）

在 **规范已 LOCK** 的前提下，适合 Ralph 的任务（**不适合** 改 EXPERT_LOCK 剧情）：

| 任务 | Ralph 适合？ | 完成标准示例 |
|------|:------------:|--------------|
| G-SHOT-T A002–A005 Shot Map 草稿 | ✅ | 五案均有 `### SC-XX` + §11 字段 checklist 全绿 |
| 读者群反馈 markdown 汇总 | ✅ `/loop` 30m | 新 feedback 文件聚类进索引 |
| 日文 E04 批量标注 | ⚠️ HITL 先 | 抽检 3 段无直译腔 |
| 重写 A001 失声机制 | ❌ | 人类 + EXPERT_LOCK · 不用 AFK Ralph |

**示例 prompt 片段（Shot Map 批量）**

```markdown
Task: 为 A002 写 V2 Shot Map，链 正式版/01_正文/案02_*.txt SC 段落。
Constraints: 复用 16_V2空间资产复用 doc 教室/侧廊，不重画建筑。
Verify: 文件存在且含 P0 节点≥6。
Done: <promise>COMPLETE</promise>
Max: 15 iterations in worktree.
```
