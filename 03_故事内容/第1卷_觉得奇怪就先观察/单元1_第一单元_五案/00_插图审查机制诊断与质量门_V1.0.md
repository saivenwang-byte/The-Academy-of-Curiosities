## 一、你的问题：规范有了，为什么还错？

**有规范，且编+导+设计分工清晰。** 失效的不是「有没有 brief」，而是 **审查停在了文字层，没有绑定成图像层**。

| 层级 | 已有规范 | 以前审查验了什么 | 以前**没**验什么 |
|------|----------|------------------|------------------|
| 编+导 | 分镜文字稿 · 画面瞬间 · 取景 · G-CAST | G-BRIEF PASS | 成图像素 |
| 译部 | JP 锚句 · B 轨日文 | translation PASS | 成图是否含中文 |
| 设计部 | Style B · L0 服色 · §11 | prompt 含 MAX_BODIES 字样 | 发色/葛西/墙牌 |
| 导演组 | G-CAST 审定表 | （多为 `[ ]` 未签） | COUNT_PASS 人头 |

**A001 StyleB 批量出图事件（2026-06-10）根因链：**

```
文字 brief PASS → produce preflight exit 0
    → Agent 一次 GenerateImage ×7（无逐帧 post 审计）
    → 无 manifest 晋升门 → 全部写入 StyleB_V3.9/
    → 用户肉眼才发现：灰发 / 学堂趣事录 / 葛西名牌
```

**结论：审查对「文字 SSOT」部分有效；对「成图 SSOT」此前无效。**

## 三、新质量门（三道 · 禁止跳步）

### 门 1 · 出图前（文字）

```bash
python tools/workflow_preflight.py --mode produce --phase generate-image \
  --prompt A001/02_分镜头/prompts/DA2.md --case A001
```

**必过**：编+导 PASS · 译部 PASS · `g_cast_prompt_gate`（含 `00_PROMPT_HARD_LOCK_V1.0.md` 引用 + 发色 + 禁中文/葛西）

**规则**：**一帧一跑** · 禁止 `--all` 批量 GenerateImage

### 门 2 · 成图后（像素 · 人工+清单）

1. 对照 [`academy-visual-auditor`](../../../../.cursor/skills/academy-visual-auditor/SKILL.md) P0 清单目检  
2. 填写 verdict → `illustration_canon_manifest.json`

```bash
python tools/illustration_canon_manifest.py --set DA2 A001_DA2_...png REJECT 0
python tools/illustration_canon_manifest.py --set DA4 A001_DA4_...v1.1.png PASS 1
```

**verdict**：`PASS` | `REVISE` | `REJECT` · 仅 `PASS + canonical=true` 可晋升

### 门 3 · 晋升正本 / 绑 PDF

```bash
python tools/workflow_preflight.py --mode produce --phase post-image --shot DA4 --case A001
python tools/quarantine_non_canonical_illustrations.py --case A001
```

**deliver 试读** 另需：M0-B · G-AB-JP · COUNT_PASS（不变）

## 四、量产（A002–A005）执行顺序

```
1. 复制 A001 门 1–3 脚本与 manifest 模板
2. 每案：STYLE DEMO 单帧 PASS → 再 P0 逐帧
3. 每帧：generate-image → 审计 → manifest → post-image → 仅 PASS 留 StyleB_V3.9/
4. 全案 P0 canonical 齐 → deliver review-pack
```

**算力纪律**：REJECT/REVISE **不得**进审阅包 · 自动 `_quarantine/`
