## 一、现流程哪里违反了您的逻辑

### 1. 「规范 9.8」在「体验 7.1」之前就签了 G-EDITOR

GPT_LOCK 自评 **9.8** 量的是场结构、FC 标注、EXPERT_LOCK —— 问的是「制作规范有没有做到位」。  
doc 22 读者群均 **7.1** 量的是「四年级能不能独立读、会不会被 `### SC-01` 吓跑」。  
**2026-06-08 已签 G-EDITOR**，等于在读者/专家体验 **未达标** 时，把正文当成了「可下游消费」的基线。

### 2. Reader Voice 在满意度过关 **之前** 就派生了

[`00_ReaderVoice派生记录_V0.1.md`](../正式版/02_读者正文/00_ReaderVoice派生记录_V0.1.md) 记录：G-EDITOR 签后 **机械派生五案 Reader**。  
按您的逻辑，Reader export 应在 **专家 ≥9.8 / 卷 ≥9.5 / 读者无 P0 跳读** 之后。  
当前读者分 **7.1**、P0 跳读（SC 场头、A002 中段）仍在 —— **顺序错了**。

### 3. KF-LOCK-T 与 G-EDITOR 同步签 · 抢在正文深度之前

37 帧 P0 brief 已 IP 签 —— 但 doc 23 仍列 A002 机制拖、A004/A005 padding、科学 P0 pending。  
**关键帧文字锁** 应跟 **G-BODY（正文过关）** 走，不应跟 **规范自评** 走。

### 4. 双轨 EDITOR/Reader 把「写作」拆成两条线

写作期维护 `HybridVoice` + 规划/派生 `ReaderVoice`，Agent 容易在 **正文还没深 enough** 时去做 strip/派生。  
您要的是：**一条 CN_BODY 写到够深 → 评审 → 过关后一键 export**。

### 5. 门禁太多、且串在正文之前

G-EDITOR → G-CN → KF-LOCK-T → G-SHOT-T → G-JP → G-IMG —— 正文期就要过 **6+ Gate**。  
您的路径是 **4 段**：标题 → 主干 → 正文 → **评审循环** → （过关后）**一个生产化包**。

## 二、建议改什么

### 规则层

| 改什么 | 怎么做 |
|--------|--------|
| **合并 G-EDITOR + G-CN** | 正文期只有一个 **G-BODY** · G-CN LOCK **挪到过关后** |
| **新 Agent 规则** | [`.cursor/rules/02-cn-body-minpath.mdc`](../../../../.cursor/rules/02-cn-body-minpath.mdc) — 正文期 **禁止** 跳过 REVIEW 派生 Reader / 签 KF / 开 G-JP |
| **两轨说明** | 正文期 **单轨 CN_BODY** · Reader = 生产化 export 选项 |

### 专家 / 读者层

| 改什么 | 怎么做 |
|--------|--------|
| **阶段从 N 个收到 2 个** | ① 正文评审（专家+读者 **同轮**）② 生产化评审 |
| **打分格式不变** | 仍用 doc 22 定量 + doc 23 好坏清单 · **必须给建议** |
| **科学/E07** | REVIEW 轮只 **标注 PENDING** · 人类签 **进生产化包** · 不阻塞正文小改 |

### 物质阶段（文件/Gate）

| 阶段 | 做什么 | 不做什么 |
|------|--------|----------|
| **正文期** | 改 `CN_BODY` · 跑 REVIEW LOOP | Reader 派生 · KF-LOCK · G-JP · 正式插画 |
| **过关后** | G-BODY 签 → §5 生产化包一次性开 | 不再重复「体验向」正文大审 |

## 三、若采用最小路径 · 下一周只做 3 件事

1. **REVIEW LOOP 第二轮（正文小改 + 同轮复评）**  
   - 按 doc 23 优先修 P0：SC 场头/metadata · A002 展示膜重复 · 生产术语叙事化  
   - 出 doc 22/23 **同 revision** 复评表 · 目标：专家案均 **≥9.5** 起步、读者 **无 P0 跳读**

2. **冻结一切生产化动作**  
   - 不新开 G-JP · 不 LOCK G-SHOT-T · 不修订 KF-LOCK-T · 不更新 Reader 派生  
   - 允许 **G1DRAFT** 内部探索图（非 PRODUCT）

3. **过关线达标后 · 开一次生产化包**  
   - IP 签 **G-BODY** → 并行 **G-JP** + **G-KF-T** → **KF-LOCK-J** → **G-KF-REVIEW** → G-IMG（见 [`26_插画门禁路径判断…`](./26_插画门禁路径判断_正文后关键帧_vs_日译优先_V0.1.md)）
