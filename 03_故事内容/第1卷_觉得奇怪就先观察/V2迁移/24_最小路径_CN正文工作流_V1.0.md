## §0 原则：一条正文线 · 评审前置 · 生产后置

| 原则 | 含义 |
|------|------|
| **一条正文线** | 正文期只维护 **CN_BODY** 一层；不并行 EDITOR/Reader 双轨写作 |
| **评审前置** | 专家 + 读者 **同轮** 打分与小改，**先于** 一切派生/签核/生产化 |
| **生产后置** | G-JP · KF-LOCK-T · G-SHOT-T · G-IMG · Reader export **全部** 在 **G-BODY** 之后一次性解锁 |
| **深度优先** | 正文未达过关线前，**禁止** 以「规范全绿 9.8」替代「出版体验 ≥9.5」 |

**最小路径总图**：

```
TITLE → SPINE → BODY → REVIEW LOOP（专家+读者同轮 · 小改迭代）
                              ↓ 过关
                    PRODUCTION PACK（后置 · 一次性）
```

## §1 四段最小路径（TITLE → SPINE → BODY → REVIEW LOOP）

### 1.1 TITLE（标题）

| 项 | 要求 |
|----|------|
| **输入** | Case Card · 锐利五要素 · [`02_标题命名逻辑_锐利五要素_V0.1.md`](./02_标题命名逻辑_锐利五要素_V0.1.md) |
| **输出** | 案标题 LOCK（可微调 · 不阻塞 SPINE） |
| **谁审** | IP Owner 或主编 **一眼过** · 无独立 Gate |
| **禁止** | 标题未定时批量开正文 |

### 1.2 SPINE（主干）

| 项 | 要求 |
|----|------|
| **输入** | 标题 + Case Card · 分场脚本 V0.2 |
| **输出** | 场序 · FC 清单 · 八秒句 · 尾钩 · EXPERT_LOCK 约束 |
| **谁审** | 叙事席 + 科学席 **结构扫**（≤30min/案）· 无分数 Gate |
| **禁止** | SPINE 未定 → 写满篇正文 |

### 1.3 BODY（正文 · CN_BODY）

| 项 | 要求 |
|----|------|
| **输入** | SPINE + academy-engine → academy-voice-editor 润色 |
| **输出** | `正式版/01_正文/案0X_*_CN_BODY_V2.0.txt`（见 §2 命名迁移） |
| **写什么** | Hybrid Voice · 陸珣声线 · 细节 + 科学可跟做 · **读者可读的叙事**（场标/FC 标签 **可内嵌写作期保留** · 评审后统一 strip） |
| **谁写** | 创作 Agent + voice-editor |
| **禁止** | BODY 未完成 → 开 REVIEW 以外的任何 Gate |

**写作期质量自检**（非 Gate · 进 REVIEW 前自扫）：

- academy-engine L2 规范项
- academy-literary-audit 场级 20 项
- 科学机制与 FC 在叙事中 **可见**（非仅标签）

## §2 单一正文文件（CN_BODY 一层）

### 2.1 合并 EDITOR / Reader 为 CN_BODY

| 旧轨 | 新口径 |
|------|--------|
| `HybridVoice_V2.0.txt`（EDITOR） | **CN_BODY 主文件** · 正文期唯一 SSOT |
| `ReaderVoice_V2.0.txt` | **生产化选项** · G-BODY 后 **一键 export**（见 §5） |

```

正式版/02_读者正文/案0X_<标题>_ReaderVoice_V2.0.txt  ← G-BODY 后 export · 可选
```

## §3 评审席（谁、何时、评什么）

### 3.1 何时

| 时机 | 动作 |
|------|------|
| CN_BODY 初稿齐 | **第一轮** 联合打分 |
| 每轮小改后 | **复评** · 直至 §4 过关 |
| G-BODY 后 | **不再** 重复正文体验评审 · 转入生产化评审（§5） |

### 3.2 谁（同轮 · 必须同时出分）

| 席 | 角色 | 评什么 |
|----|------|--------|
| **叙事** | 轻推理编辑 | 钩子 · 公平 · 情绪/洗冤 · Hybrid Voice · 节奏 |
| **科学** | 机制顾问 | P06 可跟做 · FC 与机制一致 · 人类签 **状态标注** |
| **E07** | 校园文化 | 名古屋校园 · 室内鞋/公开日/失物柜 · **预判** |
| **E04** | 日译预判 | 句长 · 术语日文化难度 · **不阻塞 CN** |
| **E06** | 分镜预判 | 深度锚点可画性 · FC 可见度 |
| **读者群** | R1/R2/R3/P1/T1 | 钩子 · 公平 · 情绪 · 可读 · **P0 跳读点** |

### 3.3 评什么 · 交付格式

**定量**：[`22_V2单元1五案_专家组读者群联合打分_V0.1.md`](./22_V2单元1五案_专家组读者群联合打分_V0.1.md) §1 逐案表

- ✅ 好在哪里（引用 SC/句）
- ❌ 不好在哪里
- 为什么扣分
- 建议怎么做
- 读者群 persona 原话

**同轮硬规则**：

- 专家分与读者分 **同一 revision** · 禁止「先 LOCK 9.8 再补读者试读」
- 科学/E07 **可标 WORKING/PENDING** · 但须在好坏清单 **显式列出** · 不隐藏到生产期

```
  → G-KF-REVIEW（主编+E06-S+auditor+IP · G-SHOT-T LOCK · §5 清单）
```
