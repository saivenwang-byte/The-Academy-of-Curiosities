## 2. 三 P0 可自测维度（H / C / S）

### 2.1 幽默 Humor (H)

| 检查项 | 可测标准 | 证据写法 |
|--------|----------|----------|
| **Running gag** | 每案 ≥1 贯穿小癖 · 重复 ≥2 次 · 读者可预期 | 「志郎：更牢/未登记/偏了一格/洗不掉」 |
| **笑点密度** | 目标 **每 ~800 字 1 个可出声笑点**（序+场1 至少 2 处） | 行号 + 引句 |
| **志郎型喜剧结构** | 动手快一步 → 越修越糟 / 字面误会 / 对照实验失败但认真 | 与机制 **同帧** 不抢 FC |
| **禁** | 恐怖化 · 嘲笑受害者 · 说明式笑话 | — |

**自评 0–10**：  
- 0–3 = 无 running gag · 全文无笑  
- 4–6 = 1 处小梗 · 不可期待  
- 7–8 = gag ×2+ · 密度达标 · **仍须 E20-H1**  
- 9–10 = **禁止自评**（仅 E20 孩子 ≥4/5 可议）

### 2.2 可爱 Cute (C)

| 检查项 | 可测标准 | 证据写法 |
|--------|----------|----------|
| **渴望+缺口** | 核心四人各 1 句 **想要什么 + 缺什么**（非功能标签） | 光：想被信却说不出 |
| **可画外形** | 每人 ≥1 **服道/习惯动作** 可进 L0（doc35） | 橙背心/银框镜/绿格/撞门框 |
| **Mascot 候选** | 标记 1 人/物为 **破圈种子**（瑆/社徽/翘角） | 翘角+观察眼社徽 |
| **禁** | 仅写「记录员/动手党」功能位 | — |

**自评 0–10**：  
- 0–3 = 四人同质 · 无外形记忆点  
- 4–6 = 功能清晰 · 缺口模糊  
- 7–8 = 渴望+缺口+可画 **齐** · mascot 轻埋  
- 9–10 = **禁止自评**（须 E20 角色记忆 + 「想加入」探针）

### 2.3 表层简单 Surface (S)

| 检查项 | 可测标准 | 证据写法 |
|--------|----------|----------|
| **Hook 前 150 字** | 异常+被指/不可能 **已落地**（非预告） | 冷开广播/对不起/空海报/封条抽屉/无影 |
| **序无卷弧剧透** | 序章 **无**「五案拼图/他还不知道」类预告 | grep 负例清单 |
| **开篇有名角色 ≤4** | 序+场1 前 800 字内 **≤4** 个有名角色登场 | 光/珣/慧美/志郎 · 水野冲突点入 |
| **一机制前景** | 单案序+场1 只 **前景化 1 个** 怪事机制 | 不编织五案网 |
| **禁** | 转学事务长铺 · 学科/引擎外溢 | — |

**自评 0–10**：  
- 0–3 = 钩子第 4 场后 · 序章剧透  
- 4–6 = 有钩子但过载 · 角色 >4  
- 7–8 = 150 字 hook · ≤4 人 · 无剧透  
- 9–10 = **禁止自评**（须 E20 续读 + 八秒句复述）

## 3. 自评模板（诚实 · 非 fake 9.5）

```yaml
case: A00X
revision: R16_globalIP
date: YYYY-MM-DD
humor_H:
  score: 0-8          # 封顶 8 · 9+ 仅 E20
  evidence:
    - "行NN: 引句"
  running_gag: "案别 gag 名"
  laughs_per_800w: N
cute_C:
  score: 0-8
  evidence:
    - "角色: 渴望+缺口"
  mascot_seed: "瑆/社徽/…"
surface_S:
  score: 0-8
  evidence:
    - "前150字: …"
  opening_named_chars: N   # 目标 ≤4
  arc_spoiler_in_xu: false
composite_note: "H/C/S 自评 ≠ G-BODY PASS · 不替代 E20"
verified: pending_E20
```

**硬规则**：

- 单项自评 **>8 须标注**「待 E20 / 待孩子」  
- **不得** 因自评上调 `scores_mvp_latest.json` 的 `卷专家/卷读者`  
- 工具输出写入 `global_ip_surface_self` · **与** `global_ip_expert` **并列**
