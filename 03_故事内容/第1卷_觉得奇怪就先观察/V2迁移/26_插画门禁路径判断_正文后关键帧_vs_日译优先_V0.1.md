## §1 结论（给 IP Owner 一段话）

**推荐采用「并行混合路径」——不是纯 Flow A（先关键帧后日译），也不是纯 Flow B（日译完再关键帧）。**

G-BODY 过关后，**G-JP 全案** 与 **G-KF-T 构图锁（CN interim 字）** 同时开跑，可节省 **2–4 周墙钟**；二者在 **KF-LOCK-J** 汇合（只 diff 画内字列，不重开构图），再经 **G-KF-REVIEW** 统一过主编/E06-S/visual-auditor/IP 与 doc 21/07 规范，最后才 **G-IMG** 出 PRODUCT PNG。

纯 **Flow B（日译后再关键帧）** 不是最短路径——日译门禁（E04+E07+读者 JP+篇幅改写）通常占 **3–5 周/单元**；若等 JP 全过再写 37 帧 brief，G1 灰模与画师排期整体后移。**但** Flow B 对 **字重帧**（A001 DA3 `0328` 屏显、A002 黑板「对不起。」、A003 壁报四栏、A005 DB1 三格 SUM）**返工最少**——画内 JP 一次定稿，画师不必按 CN interim 排版后再改气泡宽度。

### 1.1 目标 × 更短路径（赢家表）

| 目标 | 更短路径 | 理由 |
|------|----------|------|
| **最快看到构图 / G1 灰模** | **混合路径 · 并行 B（G-KF-T）** | G-BODY 后 **0 等待** 即可锁构图/站位/公平线索；不必等 G-JP 全案 |
| **最少返工（气泡 / 屏显 / 黑板 / DB1）** | **纯 Flow B**（或混合路径的 **KF-LOCK-J 汇合点**） | 画内字以 **JP 定稿** 为 SSOT；T 阶段仅 CN interim + `INTERIM` 标注 |
| **最快出版级 PNG（PRODUCT v1.0）** | **混合路径（并行 + G-KF-REVIEW）** | 墙钟 = max(G-JP, G-KF-T) + 字列 diff + 一次帧审；比串行 A 或 B **各少 1 个完整阶段等待** |
| **最低执行难度（团队少协调）** | **纯 Flow A** | 单语、单序；但 **PRODUCT 阶段返工风险高** · 不推荐作 canonical |
| **首市场 JP 试读就绪** | **混合路径** | G-JP 与插画准备 **并行** · G-MVP 仍须 **G-JP + G-IMG** 齐 |

### 1.2 推荐 canonical 序列（正典顺序）

```
G-BODY（CN REVIEW LOOP 过关 · 专家案≥9.8/卷≥9.5/读者≥95%无P0）
  │
  ├─ 并行 A ── G-JP 全案
  │              E04 日译 → academy-jp-voice-editor → E07 田中
  │              → 项目 JP Gate（doc 14 §3 · 0 实质红项）
  │              → 读者 JP 试读润色（可选 E20）
  │
  └─ 并行 B ── G-KF-T 构图锁
                 Shot Map V2 草案 + KF_LOCK_T brief
                 CN interim 画内字 · 标 INTERIM · 无 PRODUCT PNG 承诺
                 允许 G1DRAFT 灰模探索（visual-auditor 逐帧 · 非 PRODUCT）
  │
  ▼ 汇合
KF-LOCK-J（画内 JP 字列 diff → brief v0.2 · 不重开构图）
  │
  ▼
G-KF-REVIEW（主编 + E06-S + visual-auditor + IP · §5 清单 · doc 21/07/16）
  │            含 G-SHOT-T LOCK（分镜文字定稿 · 原 doc 14 独立 Gate 并入此步）
  ▼
G-IMG（插画组 PRODUCT v1.0 · depth_anchor PNG · auditor PASS）
  │
  ▼
G-MVP（E22 排版 · JP 试读包）
```

**为何纯「日译后再关键帧」不是最短，却是字重帧最低返工？**

| 帧类 | 代表 | 若 T 阶段用 CN interim | 若等 G-JP 再 brief |
|------|------|------------------------|---------------------|
| 屏显 / metadata | A001 DA3 `排练_0328.wav` · A005 metadata 三帧 | 日文栏宽/假名密度不同 → **PRODUCT 须改 UI** | JP 字锁后再排版 → **一次成型** |
| 黑板 / 壁报誊清 | A002「对不起。」· A003 四栏 · A005 五栏章程 | 笔画数/竖排占位估不准 | 誊清字与 JP 句长对齐 |
| DB1 机制 SUM | 五案 B 轨三格简图 | CN 简图 ≠ JP 机制句 | **KF-LOCK-J 专责** · 见 doc 21 §1 |

→ **混合路径**：构图/站位/线索在 **B 并行** 先锁；字重元素在 **汇合点** 按 Flow B 标准处理——**兼得墙钟与低返工**。

## §2 Flow A vs Flow B 对比

**Flow A（CN 先关键帧）**：G-BODY → KF-LOCK-T / G-SHOT-T → 帧审 → G-IMG（CN interim 字）→ 后补 G-JP → KF-LOCK-J 改字  

**Flow B（JP 先再关键帧）**：G-BODY → G-JP 全案 → KF-LOCK-T+J 合并 → 帧审 → G-IMG  

**混合（推荐）**：G-BODY → **并行** G-JP + G-KF-T → KF-LOCK-J → G-KF-REVIEW → G-IMG  

评分 **1=最优/最易 · 5=最差/最难**（数值越低越好）。

| 维度 | Flow A | Flow B | 混合（推荐） |
|------|:------:|:------:|:------------:|
| **墙钟时间** | 2 | 4 | **1** |
| **PRODUCT 返工风险** | 4 | **1** | **2** |
| **执行难度（协调成本）** | **2** | 3 | 3 |
| **G1 灰模可见速度** | **1** | 5 | **1** |
| **首市场 JP 对齐** | 4 | **1** | **2** |
| **与 doc 24 最小路径一致** | 3 | 3 | **1** |

**一句话**：Flow A **快启动、慢收尾**；Flow B **慢启动、稳出图**；混合 **正典**。
