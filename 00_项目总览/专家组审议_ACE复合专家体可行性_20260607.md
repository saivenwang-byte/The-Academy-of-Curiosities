## 一、五方专家组表决

> 表决选项：**可行** / **部分可行** / **不可行**  
> 本审议基于用户方案：公开 IP 生产史研究 → 能力原子 → 复合阶段专家；RAG + 规则 + 正典门禁；**禁止**具名真人 Agent / 数字人。

### 1.1 IP 创作 / 品牌（E01 · IP Owner 视角）

| 项 | 表决 | 理由 |
|----|:----:|------|
| ACE 四层与 Vol1 阶段对齐 | **可行** | Gate A/B/C/D 与现有 PRODUCT-GATE、四栏分发、试读出厂线一一映射；不另起平行流程 |
| 复合专家替代「田中みどり式角色 Agent」 | **部分可行** | 田中 HTML 工具可保留为 **E07 校准 UI**；ACE 负责 **阶段编排与验收清单**，不抢 IP 否决权 |
| 全球 IP 对标驱动 Vol1 差异化 | **可行** | 已有 [`竞品参照分析_…V1.0.md`](./竞品参照分析_放課後ミステリクラブ与全球桥梁书IP_V1.0.md)；ACE 把对标从「文档」升级为「可执行检查项」 |
| 首建 ACE-B（样章视觉） | **可行** | 当前瓶颈在 Gate A→B：L0 六帧、G1 PNG、depth 锚点、Shot Map — 与 ACE-B 职责完全重合 |

**IP 方小结**：**部分可行 → 建议启动 ACE-B V0.1**；ACE-A/C/D 待 B 回测后再扩。

### 1.2 出版 / 编辑（E04 · E16 · E22）

| 项 | 表决 | 理由 |
|----|:----:|------|
| ACE 不替代 E04 硬门禁 | **可行** | 用户方案明确 E04 每案必签；ACE 仅产出 **待签草案 + 差异表** |
| 蒸馏「出版生产史」而非「编辑个人风格」 | **可行** | 与 [`13_V1.1` §0](./13_真实世界人物专家蒸馏库_V1.1.md) 一致：提取机制，不复制文体 |
| ACE-B 覆盖 Shot Map / 插图深度 / 版式组合 | **部分可行** | E22 五层排版 LOCK 已有；ACE-B 应 **引用** E22 正典，不另写版式宪法 |
| 虚拟专家增补计划（佐藤健一等具名 Agent） | **不可行**（本方案下） | 与用户「禁止某某 Agent」冲突；[`虚拟专家增补计划_20260607.md`](./虚拟专家增补计划_20260607.md) 中 P0 五条 **改挂 ACE 能力原子**，不保留假名 Agent |

**出版方小结**：**部分可行** — ACE-B 定位为 **样章视觉生产官**，E04/E22 正典为上位法。

### 1.3 法务 / 合规（E25 · 未 onboard）

| 项 | 表决 | 理由 |
|----|:----:|------|
| 研究公开 IP 生产史（访谈/报道/出版社物料） | **部分可行** | 仅限 **已公开、可溯源** 材料；禁止爬取未授权内页全集做训练 |
| RAG + 规则 + 正典门禁（非微调 scraped 内容） | **可行** | 降低著作权与「衍生训练」争议；蒸馏卡须标注 `source_type` + URL/ISBN |
| 禁止人格冒充 / 挂名 / 「石黒圭 Agent」 | **可行** | 与 [`13_V1.1` §0](./13_真实世界人物专家蒸馏库_V1.1.md) 及日本不正竞争防止法精神一致 |
| 真人姓名作「知识源引用」 | **部分可行** | 允许在蒸馏卡 `knowledge_sources[]` 列出 **公开方法论来源**（如装帧编辑公开讲座）；禁止 UI/对外文案暗示 endorsement |
| 四分离（见 §五）落地 | **部分可行** | 需 E25 正式 onboard 前由 IP Owner 暂代法务签 **Phase 0–3 合规声明** |

**法务小结**：**部分可行** — 架构正确；**启动条件** = 四分离清单全绿 + 蒸馏卡 schema 强制字段。

## 三、风险矩阵

| 风险 | 等级 | 触发场景 | 缓解 |
|------|:----:|----------|------|
| **侵权** | 高 | 爬取竞品全书 PDF/内页做语料；复制 Gurin 画风 | 仅公开访谈/报道/封面/目录级物料；`source_license` 字段；视觉 **机制** 不学 **笔触** |
| **人格冒充** | 高 | 对外称「石黒圭审阅」；UI 头像化真人 | 四分离清单 §五；对外仅「ACE 样章视觉生产官」；knowledge_sources 小字出处 |
| **平均化庸才** | 中 | 多 IP 检查项投票取交集 → 无辨识度 | **正典 veto**：与 LOCK/V12–V14 冲突时对标项自动丢弃；每原子标 `canon_priority: high|low` |
| **正典漂移** | 中 | ACE 草案绕过 Gate 直接 merge | ACE 输出仅 `draft/`；merge main 仍走 E06 + 三问检查单；`pre_push_check` 增 ACE lint |
| **Gate 双轨** | 中 | ACE-B 与 Gate A P0 抢资源 | 并行轨：P0 真人签核 **不阻塞** ACE 研究轨；ACE **不替代** G1/E04 签字 |
| **scope 膨胀** | 中 | ACE-A/C/D 同步开建 | V0.1 **仅 ACE-B**；A/C/D 写 roadmap 不建 Skill |
| **假专家依赖** | 低 | 真人长期不回传 → 过度依赖 ACE | ACE 标注 `confidence: inferred`；硬门禁位永 👤 |

## 五、合规四分离检查清单

> 启动 ACE-B V0.1 前，IP Owner 勾选；E25 onboard 后转 E25 维护。

| # | 分离项 | 要求 | 验收 |
|---|--------|------|:----:|
| S1 | **身份分离** | 对外不出现真人姓名 Agent；UI/Skill 名 = ACE-*_Officer | ⬜ |
| S2 | **知识分离** | 蒸馏卡区分 `canon_ref` / `benchmark_ref` / `knowledge_source`；禁止混为「专家意见」 | ⬜ |
| S3 | **决策分离** | ACE 输出文件头 `STATUS: DRAFT · NOT SIGNOFF`；👤 签核位独立 PDF/MD | ⬜ |
| S4 | **数据分离** | 无 scraped 全书入库；RAG 索引仅 `docs/ace-distill/` 已标注来源；gitignore 原始 scrape | ⬜ |
| S5 | **版权分离** | 蒸馏卡 `implementation: rag_rules_only`；禁止 fine-tune 第三方文本 | ⬜ |
| S6 | **视觉分离** | SC prompt 审计禁「in the style of [living artist]」；仅机制（构图/线索可见/比例） | ⬜ |
| S7 | **溯源分离** | 每条能力原子 ≥1 公开出处（URL/ISBN/访谈日期） | ⬜ |
| S8 | **漂移分离** | ACE 建议与正典冲突 → 自动 HOLD + 引用 LOCK 文件 | ⬜ |

## 六、专家组结论

### 6.1 是否启动 ACE-B V0.1？

**表决：YES — 有条件启动**

**条件（全部满足后 Week 1 开工）**：

1. IP Owner 勾选 §五 四分离清单 S1–S8（可先 S1–S6，S7–S8 在 Phase 2 完成）
2. 废弃新增 `academy-expert-<假名>` Skill；统一 `skills/ace-experts/`
3. P0 真人轨并行：E04 / E07 / E09 / G1 截止 **不因 ACE 延后**
4. ACE-B V0.1 交付 = A001 回测报告 + 1 份 Officer SKILL + ≥8 张蒸馏卡
5. 6 周 sprint 结束 IP Owner 裁决：扩 ACE-A 或 hold

### 6.2 与用户一句话对齐

> **学全球 IP 怎么把一本书做出来，不学任何人的人设；ACE 是阶段生产官，不是明星数字人；真人签字的地方，ACE 只递文件夹。**
