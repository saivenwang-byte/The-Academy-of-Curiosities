## 0. 什么叫「完整产品」（给编辑·主编·美术·专家的一课）

**完整产品 ≠ 一个文件 · 一个 prompt · 一张图。**

完整产品 = **某一出版阶段结束时，下游角色（试读组织者 / 出版社责编 / 印厂 / 渠道）可以零追问接手** 的 **文件包 + 构建 reproducibility + 门禁对照表 + 已知缺口清单**。

| 错误做法（禁止） | 正确做法 |
|------------------|----------|
| 「正文好了，要不做插图？」 | 按 **P1/P2/P3** 清单 **整包齐** 再报 |
| 「PDF 有了，问卷你自己想？」 | P1 必含 **脚本+问卷+flyer+PDF+PPT** |
| 「Skill 改了两处，还要改 CLAUDE 吗？」 | P0 入口 **整表同步** · 自审脚本 PASS |
| 「你觉得试读阈值够吗？」 | **E20 阈值写死在包内** · 试读后只回流数据 |
| 每步问用户「继续吗？」 | **缺口写进 `门禁对照表` 的 ⬜/🟡** · 不阻断已齐项 |

**Agent 汇报格式（固定）**：

```
阶段：P? · 包名
自审：audit_phase_package.py → PASS/FAIL
专家轨：🤖 已跑项 / 👤 待人审项（列编号）
交付路径：（列表）
缺口：（仅列 ⬜/🟡 · 不写「要不要做」）
```

**只有以下情况才问用户**：IP 委员会级决策（增删人物、改 Vol1 结构、换首市场、签约边界）— 见 [`F1`](../00_项目概览/01_分支讨论_范式与框架/10_IP战略参考_ChatGPT对话/F1_Vol1结构与隐藏线.md)。

## 7. Agent 自审协议（硬规则）

### 7.1 会话内流程

```
1. 用户给目标阶段码（或从上下文推断：试读=P1 · 送社=P2）
2. 读本文 § 该阶段完整清单
3. 缺项 → 补项（不 ask 用户「要不要补」）
4. 跑 audit_phase_package.py
5. 跑 11_检查项 该阶段子集（见脚本输出）
6. 输出 §0 固定汇报格式
7. 仅 ⬜ 且属 IP 委员会决策项 → 标注「待委员会」· 不问「继续吗」
```

### 7.2 调用的 Skill（按序）

| 顺序 | Skill | 阶段 |
|------|-------|------|
| 1 | `academy-series-architect` | 范围是否越界 |
| 2 | `academy-engine` | C03/P 类 |
| 3 | `academy-jp-voice-editor` | P1/P2/P3 日语文本 |
| 4 | `academy-visual-auditor` | V13/V14 |
| 5 | `audit_phase_package.py` | 文件存在性 |

### 7.3 禁止问用户的话术

- 「要不要做问卷 / flyer / PPT？」→ **P1 清单有就必须做**
- 「插图只有 placeholder 可以吗？」→ **写入 🟡 · 交付其余完整包**
- 「先 commit 还是先试读？」→ **包齐+自审 PASS 再 commit**
- 「下一步你想做哪个？」→ **写「建议下一阶段：P?」· 不阻塞当前包**
