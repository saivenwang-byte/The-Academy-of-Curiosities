# 文档日期语义说明 · 2026-06-14

> **问题**：仓库文件名/正文含 `20260612`–`20260613` 等日期，Git 提交可能发生在 **2026-06-07** 前后。  
> **风险**：未标注时，读者误以为「未来日期的 human PASS 已真实发生」。  
> **Authority**：IP Owner · 与治理冻结令 §1.4 Truth 线一致

---

## 1. 日期类型（写入元数据时必须选一）

| 类型 | 标记 | 含义 |
|------|------|------|
| **actual_completion** | `Status: COMPLETED · actual · YYYY-MM-DD` | 工作已完成 · Git/人工可核对 |
| **planned_target** | `Target: YYYY-MM-DD · not_actual` | 计划截止 · **未完成** |
| **batch_label** | `Batch N · label date YYYY-MM-DD` | 批次文件名 · 非完成证明 |
| **simulated** | `SIMULATION · not E20` | 合成读数/LLM 面板 |

## 3. 新文档规则

1. 文件名日期 = **批次/主题** 时，文首加：`> **Date in filename**: batch label only`  
2. human PASS 必须：`签字人 · actual · YYYY-MM-DD · 证据路径`  
3. Agent 产出禁止写「已于 MM-DD 完成专家签字」  
4. 对外材料只用 **actual_completion** 或删日期
