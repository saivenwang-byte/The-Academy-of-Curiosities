# 单案故事引擎 V2.0 · 执行规范

> 状态：`MIGRATION_SPEC`  
> 目标：将现有“趣事—观察—验证—温柔真相”升级为“强异常—高压误判—跨学科取证—责任澄清—关系修复”。

## 1. 单案标准流程

```text
01 强异常画面/声音
02 第一误判迅速成形
03 被指向者开始付出代价
04 主倒计时建立
05 观察社分工调查
06 三至四条公平线索出现
07 朋友/班级/规则制造立场冲突
08 核心机制完成验证
09 错误责任被洗清
10 真实责任被确认
11 关系以可见动作修复
12 陸瑆记录“哥哥救回了谁/什么”
13 留下卷级关系网变量
```

## 2. 单案必须字段

每案必须明确：

- 一句话钩子；
- 强画面或声音；
- 被指向者；
- 主倒计时；
- 主压力与副压力；
- 错误责任；
- 真实责任；
- 主机制与交叉机制；
- 3—4条公平线索；
- 1条主要误导；
- 1个人物秘密；
- 1个修复动作；
- 1个卷级变量。

| 项目 | 上限 |
|---|---:|
| 主怪事 | 1 |

| 主倒计时 | 1 |

超出必须在Case Card中写明删减或拆案方案。

## 4. 三重/四重真相

每案至少完成前三层，推荐完成第四层：

1. **机制真相**：怪事如何发生；
2. **传播真相**：错误故事为何被相信；
3. **人物真相**：被怀疑者在隐瞒或害怕什么；
4. **责任真相**：他没做被指控的事，但是否需承担另一部分责任。

## 5. 角色动作分工

- 陸珣：空间、路线、结构、差异；
- 伊藤光：发现、推进、连接、可能先误判；
- 加藤慧美：记录、采访、情绪、证据校准；
- 松本志郎：实验、复现、装置、可能想复杂；
- 陸瑆：不参与破案；补足家庭、人物、情感和跨卷回声。

禁止四人站桩讨论。每案至少有三人通过不同动作取得不同类型证据。

## 6. 压力必须推动选择

压力不是背景说明，必须迫使角色做出选择，例如：

- 为保住班级资格，要不要让一个人先认错；
- 为替朋友保密，要不要隐瞒关键时间；
- 为证明清白，要不要公开对方最不想被知道的事实；
- 为维护老师或班长权威，要不要延迟公开反证。

## 7. 知识使用规则

知识只能承担三种工作：

1. 制造怪事；
2. 留下证据；
3. 在最后关头作证。

禁止：

- 先确定“本案教某知识”再造剧情；
- 两页以上连续讲解；
- 用术语替代动作；
- 人际真相替代科学谜底；
- 用科学解释洗掉人物责任。

## 8. 情绪曲线

```text
惊奇 → 轻松/幽默 → 怀疑 → 人际冲突 → 短暂喘息 → 倒计时压迫 → 验证翻盘 → 温柔余震
```

高压段连续不得超过全案约35%。每案必须有至少一个减压场景。

## 9. 结尾规则

结尾不得使用：

- 全班集体鼓掌；
- 所有人整齐道歉；
- 旁白总结正确道理；
- 老师替角色说完全部价值。

必须用一个可见的小动作收束，例如：

- 座位被重新留出；
- 名字被写回名单；
- 饭盒重新打开；
- 报名表重新贴回；
- 第一传播者主动纠正一句话。

## 10. 单案Gate

| Gate | PASS标准 |
|---|---|
| H1 钩子 | 8秒内可复述，3秒能指出异常 |
| H2 压力 | 倒计时后果明确且不可逆 |
| H3 公平 | 核心答案所需线索均提前出现 |
| H4 机制 | 可验证、可解释、无超自然兜底 |
| H5 情感 | 读者知道谁正在失去什么 |
| H6 责任 | 错误责任与真实责任分开 |
| H7 修复 | 有可见关系变化 |
| H8 卷网 | 留下1个非悬案式卷级变量 |
