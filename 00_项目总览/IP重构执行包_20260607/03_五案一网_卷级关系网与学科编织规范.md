# 五案一网 · 卷级关系网与学科编织规范

> 状态：`MIGRATION_SPEC`  
> 目标：保证五个独立小案在卷末形成结构性回收，而非简单并列或强行拼接。

## 1. 基本定义

- **案**：独立怪事、独立谜底、独立情绪闭合。
- **卷**：五案共同组成的一张关系网。
- **卷级变量**：某案结束后仍在现实中继续存在的物件、规则、信息、空间变化或人物选择。
- **卷级元问题**：五案从不同角度共同回答的一个问题。

## 2. 五案功能分配

| 案 | 卷级作用 | 情绪功能 |
|---|---|---|
| A001 | 投下第一颗石子 | 好奇、建立规则 |
| A002 | 证明影响越过原地点/班级 | 扩散、意外 |
| A003 | 进入传言、人际或信息层 | 犹豫、误解、站队 |
| A004 | 补救行为反向制造风险 | 背叛、责任、倒计时 |
| A005 | 多条关系汇合并完成整卷回收 | 全校级翻盘与修复 |

## 3. 允许的关系类型

- `causal` 因果：A改变B。
- `common_source` 共源：多个事件来自同一隐藏条件。
- `puzzle_piece` 拼图：每案提供大真相的一块。
- `mirror` 镜像：同一人性问题换不同角色重演。
- `contrast` 对照：相似现象由不同机制造成。
- `misapplication` 误用：前案正确答案在后案被错误迁移。
- `carrier` 传递：同一物件/话语/记录穿过多案。
- `information_pollution` 信息污染：传言把无关事件编成一个故事。
- `pressure_field` 共同压力场：比赛、评比、开放日等目标扭曲多案选择。
- `role_exchange` 角色换位：怀疑者与被怀疑者互换。
- `echo` 回声：后案改变前案意义。
- `spatial` 空间：同一地点在不同时间表现不同机制。
- `probabilistic` 概率汇合：多件低概率小事同刻交叠。

每卷至少使用3种关系，最多5种主关系；超过5种视为过度设计。

## 4. 五案一网硬规则

1. 每案必须独立解决主谜题。
2. A005不得否定A001—A004已经成立的答案。
3. 卷级连接不得只靠幕后黑手。
4. 前四案共留下3—4个卷级变量。
5. A005主要使用读者已经见过的变量和证据。
6. 卷级回收必须同时完成：事件回收、学科回收、人物回收。
7. A005不得靠长篇回顾补救信息遗漏。

## 5. 学科编织结构

每卷必须有：

- 5个单案主机制；
- 2—3条案间知识连接；
- 1—2次正确知识的错误迁移；
- 1个卷级元问题；
- 1次第五案多学科汇流。

### 单案结构

`1主机制 + 1交叉机制 + 1社会放大器`

### 卷级结构

`5主机制 + 3跨案变量 + 2知识误用 + 1元问题 = 1张学科网`

## 6. 学科关系类型

- **接力**：前案方法成为后案工具。
- **互证**：一个学科结论需要另一学科确认。
- **冲突**：两类证据指向不同答案。
- **误用**：正确经验被错误套用。
- **汇流**：A005必须同时使用多个学科。

## 7. 元问题示例

- 痕迹真实，是否等于责任判断正确？
- 听见某人的声音，是否等于那个人说过？
- 大家都记得，是否等于事情发生过？
- 相同现象，是否一定有相同原因？
- 多人共同造成结果时，责任怎样划分？

## 8. 卷网设计顺序

必须先设计A005，再倒推A001—A004：

```text
A005需要哪些变量/证据/关系
→ A004自然产生其中两项
→ A003建立传言或人际连接
→ A002扩散至另一信息岛
→ A001投下第一变量并建立阅读规则
```

禁止先写完五案后再强行找连接。

## 9. 卷级可视化交付

程序与策划必须输出：

- 五案节点图；
- 关系边表；
- 学科节点图；
- 卷级变量台账；
- 角色换位表；
- A005回收矩阵；
- 未回收变量清单。

## 10. 卷级Gate

| Gate | PASS标准 |
|---|---|
| V1 独立性 | 五案均可单独阅读并闭合 |
| V2 关系性 | 至少3种关系，不依赖单一因果 |
| V3 学科性 | 学科之间发生互证/冲突/误用/汇流 |
| V4 公平性 | A005核心材料前四案已出现 |
| V5 可理解 | 儿童无需记住全部连接也能读懂A005 |
| V6 惊喜 | 回收后产生“原来前面一直有用” |
| V7 简洁 | 主跨案变量不超过4个 |
