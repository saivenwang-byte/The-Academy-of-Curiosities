# 思考角度索引 · 范式讨论用透镜

> **用途**: 范式讨论时 **选主透镜**，避免一场会里角度乱跳  
> **规则**: 新增角度须写「适用 / 不适用 / 与现有角度区别」  
> **禁止**: 在本文件写具体项目任务  
> **架构参照**: 透镜 **A1–A4、L1–L4** 的操作定义见 [`05_协作计算架构_参照系.md`](./05_协作计算架构_参照系.md)

---

## 怎么用

1. 开场声明：「本场主透镜 = **A2**」  
2. 发散时若跑题，用透镜名 + **标准问法** 拉回（见各 ID 下「架构问法」）  
3. 定案后若产生 **新透镜**，按模板追加 § 末

## A · 层级透镜（Stack）— **已加强**

| ID | 透镜 | 问什么 | 不适用 |
|----|------|--------|--------|
| A1 | **机器层 vs 产品层** | 数据平面（L1–L3）未就绪时，能否对外宣称「产品成立」？ | 单篇稿修辞 |
| A2 | **讨论层 vs 执行层** | 改的是 **控制平面（路由/边界）** 还是 **数据平面（运行时文件）**？ | 已定案后的批量生产 |
| A3 | **学习层 vs 复制层** | 升格的是 **firmware 原则** 还是粘贴 **配置文件**？ | 像素级抄界面 |
| A4 | **永久记忆 vs 工作记忆** | session 清零后，从 Cache / RAM / 磁盘哪层恢复？ | 单次对话礼貌用语 |

### A1 · 架构问法

> 「若 L1 路由或 L3 隔离未 PASS，`[产品]` 对外演示是否构成 **虚报架构就绪**？」

| 机器层 READY | 产品层可宣称 |
|--------------|--------------|
| L1 路由表 + L2 可重启 + L3 默认拒绝 | 可谈「可持续生产」 |
| 缺任一项 | 仅可谈「内容样张」，不宣称「体系成立」 |

### A2 · 架构问法（**核心透镜 · 已加强**）

> 「这条 diff 属于 **控制平面** 还是 **数据平面**？」

| 控制平面（讨论层） | 数据平面（执行层） |
|--------------------|-------------------|
| 框架、边界、路由、关闭条件、学习卡片 | 正文、图、脚本 diff、lint 修复 |
| 产出 = 契约（Ring 1） | 产出 = mutable 状态（Ring 2–3） |
| **CP-1**: 不直接写数据平面 | 必须命中 L1 路由 |

**跑题拦截句**：「这是数据平面问题，本场是控制平面 — 请换区。」

### A3 · 架构问法

> 「从 `[来源]` 提取的是 **指令集扩展（原则）** 还是 **二进制 blob（配置）**？」

- 原则 → 可写学习卡片 → 可能升格 Rule  
- 配置 → 通常 **不学**，除非中性化后成模板  

### A4 · 架构问法

> 「写入 `[对象]` 应落在 storage hierarchy 哪一层？升格条件是什么？」

见架构参照系 §3.3 判据表。

## 透镜冲突仲裁（新增）

两透镜同时适用时，**优先级**：

```
Ring 0 / L3 安全  >  A2 平面  >  L1 路由  >  L2 存储  >  L4 学习
```

例：执行层任务（A2 数据平面）若触 Ring 0（F3）→ **先停**，升控制平面。
