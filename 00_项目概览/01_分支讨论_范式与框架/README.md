## 本区讨论什么

| 讨论 | 说明 |
|------|------|
| **范式** | 用什么模型理解「长期 IP + AI 协作」？例如：四层栈、门禁流水线、分支点 |
| **思考角度** | 从哪些透镜看问题？例如：机器层 vs 产品层、学习层 vs 执行层 |
| **对外学习** | 别人（ECC、gstack、obra superpowers…）做了什么？**学什么、不学什么** |
| **建立框架** | 怎么把模糊感觉变成可讨论、可定案、可迁移的结构？ |

---

## 本区不讨论什么

以下 **一律移出本区**，到各自执行目录去谈：

- 某一卷正文、插图、角色、Prompt 的具体改法  
- Vol 进度、lint 结果、某张图 PASS/FAIL  
- 品牌名、人名、4年2組 等项目正典细节  
- 「下一步写第几卷」「这张 lineup 怎么改」类执行指令  

> 本区产出的是 **「怎么想、怎么学、怎么搭架子」**；  
> 具体项目工作见 `03_故事内容/`、`07_设计原档/`、`docs/volume_*` 等。

## 讨论规则（四条）

1. **只抽象，不落地** — 可以举外部案例，不把案例直接绑死到本项目某一卷。  
2. **先框架，后实例** — 先定「层/门/范式名」，再允许举 1 个中性例子说明。  
3. **学习须写清** — 从别人家 **学什么 / 不学什么 / 为什么** 三段式。  
4. **定案写范式** — 结论进 `04_决策记录_范式层.md`；项目向清单另开目录，不进本区。
