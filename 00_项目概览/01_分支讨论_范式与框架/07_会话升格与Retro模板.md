# 会话升格与 Retro 模板 · L4 操作化

> **Status**: ACTIVE v1  
> **用途**: 把「持续学习」从口号变成 **每场可跑** 的 5 问 + Trace 格式  
> **范围**: 范式通用 · 执行向会话结束时可选跑 · 控制平面会话 **建议必跑**  
> **原则**: 对主编有利 = 少重复纠错；对项目有利 = 教训升格而非补丁堆积

---

## 1. 何时跑 Retro

| 场景 | 必跑？ | 产出 |
|------|--------|------|
| 控制平面讨论结束 | **建议** | 决策记录 + 可选 Trace |
| 数据平面会话 >45min 或 有返工 | **建议** | Trace 短表 |
| 用户明确纠正 Agent 同一类错误 | **必跑** Trace | Ring 3 队列 |
| 纯只读问答、无 Write | 可跳过 | — |

---

## 2. Retro 五问（固定 · 2 分钟）

每场结束 Agent **自答**（写入 Trace 或决策记录 §场次）：

| # | 问 | 答（一行） |
|---|-----|------------|
| **R1** | 本场主平面？ | 控制 / 数据 |
| **R2** | 主透镜？ | 如 A2、L3、D1 |
| **R3** | 关闭条件达成？ | 是/否 + 缺什么 |
| **R4** | 重复失败或用户重复纠正？ | 无 / 有 → 类别 |
| **R5** | 是否产生 **可升格** 条目？ | 无 / 有 → 类型 |

**R5 升格类型枚举**：Rule 补丁 · Skill 修订 · SOP · lint · 学习卡片 · 架构参照 § 增补

---

## 3. Trace 短表（Ring 3 队列 · 模板）

```markdown
### Trace · YYYY-MM-DD · [短标题]

- **平面**: 控制 | 数据
- **透镜**: A2 | L3 | …
- **现象**: （一次失败或纠正，客观描述，无项目真名可用占位）
- **根因（范式）**: 路由缺失 | Ring 误判 | 缓冲未读 | 透镜未用 | …
- **建议升格**: Rule | Skill | SOP | lint | 05_§ | 01_透镜 | 学习卡片
- **不学什么**: （若建议拒绝某做法，一行）
- **状态**: QUEUE | REVIEW | MERGED | REJECT
```

**规则**：

- Trace 默认 **QUEUE**（Ring 3），不直接改 Ring 2  
- **MERGED** 须控制平面 REVIEW + 主编或决策记录定案  
- 同根因 Trace **≥2 条** → 优先 REVIEW

## 4. L2 工作缓冲格式（Cache · 可选）

会话结束若下一会话可能接续，写 **≤15 行** 缓冲（路径由项目自定；范式区示例名 `working_buffer.md`）：

```markdown
# Working Buffer · YYYY-MM-DD HH:MM

## 平面
控制 | 数据

## 未完成
- …

## 下一会话第一件事
- …

## 勿做
- …

## 指针
- 决策: `04_决策记录_范式层.md` §…
- 架构: `05_` §…
```

**升格判据**（A4）：

| 缓冲内容 | 下步 |
|----------|------|
| 仅接续用 | 下会话读后 **可删** 或覆盖 |
| 含定案 | 升格 → `04_决策记录` |
| 含可复用原则 | 升格 → 学习卡片或 `05_` |

**是否进 Git**：项目自定；范式建议 **RAM 级**（决策记录摘要进 Git，缓冲可选）。

---

## 5. 升格流水线（与会话衔接）

```
会话 Retro (R1–R5)
    → Trace QUEUE (Ring 3)
    → 控制平面 REVIEW（D1 三段式若涉外部来源）
    → MERGED → Rule / Skill / SOP / 05_ / 01_
    → 下 N 场会话：R4 是否减少同类失败
```

| 升格目标 | 最低门槛 |
|----------|----------|
| `05_` / `01_` 增补 | 1 Trace + 透镜说不清的可复现例 |
| 新学习卡片 | 外部来源 + 三段式完整 |
| Rule / lint | 2 Trace 同根因 或 1 次数据平面事故 |
| 框架 v1 冻结 | SOP 五块 + E4 关闭条件 |

---

## 6. 对用户/项目的利益点

| 利益 | 机制 |
|------|------|
| **少重复纠正** | R4 强制显性化 → Trace → 升格 |
| **会话可接续** | 缓冲 + 指针，重启不从零问 |
| **补丁不堆** | Trace 队列 ≠ 直接改正典 |
| **范式区干净** | 教训升格到 05_/01_，不污染执行清单 |

## 7. 范式区 Trace 存放（本目录）

| 状态 | 路径 |
|------|------|
| 队列 | `traces/QUEUE_*.md`（按需建目录） |
| 已合并 | 决策记录 §Trace 归档 或 删除 QUEUE 留 MERGED 摘要 |

首条 Trace 可在首次 R4=有时创建 `traces/`。
