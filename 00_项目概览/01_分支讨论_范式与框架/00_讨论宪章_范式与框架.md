# 讨论宪章 · 范式与框架

> **Status**: ACTIVE  
> **读者**: 主编 + 任何参与「怎么协作、怎么学习、怎么建架子」的讨论者  
> **禁止**: 在本文件及同目录讨论中展开 **具体项目执行内容**

---

## 1. 本区的唯一任务

把 **「长期 IP + 多 Agent 协作」** 从直觉和口号，变成：

1. **可命名的范式**（我们用什么模型想问题）  
2. **可选的角度**（同一问题从哪几个透镜看）  
3. **可复用的学习法**（别人家东西怎么拆、怎么取舍）  
4. **可操作的建框架 SOP**（讨论怎么收敛成结构）

**不任务**：替具体项目做稿、改图、排期、验收。

## 2. 四层讨论栈（本区自洽模型）

本区内部用 **「讨论栈」** 组织话题，与 ECC 的 **「生产栈」** 同构但不同物：

```
┌────────────────────────────────────────────┐
│  L4 · 持续学习范式   讨论本身如何沉淀为方法  │
├────────────────────────────────────────────┤
│  L3 · 边界与安全范式  什么能谈、什么不能碰   │
├────────────────────────────────────────────┤
│  L2 · 记忆与接续范式  跨会话如何继承讨论进度 │
├────────────────────────────────────────────┤
│  L1 · 分工与路由范式  谁负责想、谁负责做     │
└────────────────────────────────────────────┘
```

| 栈层 | 在本区问什么 | 典型产出 |
|------|--------------|----------|
| L1 分工路由 | 通用 Agent vs 专精 Agent？讨论由谁主持？ | 路由原则、角色表 |
| L2 记忆接续 | 讨论断档后怎么接上？什么写进决策记录？ | 缓冲格式、升格规则 |
| L3 边界安全 | 范式讨论 vs 执行讨论的防火墙？ | 本 README 边界表 |
| L4 学习沉淀 | 一次对外学习怎么变成可复用条目？ | 学习卡片模板 |

## 5. 「对外学习」标准格式（强制）

从别人家（ECC、Cursor、gstack、novel2hermes…）学东西时，每条学习 **必须** 写成：

```markdown
### [来源名] · [能力/概念名]

**学什么**
- （可迁移的原则，1–3 条）

**不学什么**
- （场景不符、成本过高、与我们的边界冲突，1–3 条）

**为什么**
- （一句：我们缺什么 / 已有什么）

**抽象层**
- （范式名，不含具体项目名称）
```

未按此格式写的「好厉害要不要用」—— **不算本区有效讨论**，退回重写。

## 6. 「建立框架」标准格式（强制）

任何新框架提案须包含 **五块**，缺一则视为草稿：

| 块 | 内容 |
|----|------|
| **问题句** | 一句：这个框架解决什么模糊性？ |
| **边界** | 管什么 / 不管什么 |
| **分层或分维** | ≥2 层/维，每层一句定义 |
| **与外部的关系** | 借鉴谁、差异在哪 |
| **关闭条件** | 怎样算「框架 v1 可冻结」 |

详见 [`03_如何建立框架_SOP.md`](./03_如何建立框架_SOP.md)。

## 7. 讨论会话建议结构

每场范式讨论建议 **60–90 分钟等价篇幅**，固定四段：

1. **锚定**（5 min）— 读宪章 + 本场只谈哪一类（范式/角度/学习/建框架）  
2. **发散**（20 min）— 只列选项，不定案  
3. **收敛**（20 min）— 命名、分层、写学习三段式  
4. **升格**（5 min）— 写入 `04_决策记录_范式层.md` 或 SOP/索引

## 8. 升格路径（讨论 → 可复用文档）

| 讨论成熟度 | 存放 |
|------------|------|
| 草稿、有争议 | 本目录内 `.md` |
| 范式 v1 冻结 | `04_决策记录_范式层.md` |
| 可给任何长期项目用 | 升格 `docs/` 下 `00_*_FRAMEWORK.md`（另开 PR/主编批） |
| 仅历史备查 | `00_项目概览/00_归档/` |

**升格时不带项目实例**：升格版里用 `[某长期 IP 项目]` 占位，不写具体卷名。
