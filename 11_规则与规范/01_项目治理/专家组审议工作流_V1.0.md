## 0. 单行真相

| 项 | 定案 |
|----|------|
| **触发** | IP 方提出 **未决问题 / 多方案 / 边界模糊** |
| **Agent 第一步** | **先** 向 IP 方呈现结构化议题（含选项，若有）— **不** 直接写长文或改正典 |
| **第二步** | 按 E01–E25 **域分类** → 模拟 **专家小组 → 跨学科专题组 → 大专家团** |
| **第三步** | 输出固定格式（§4）→ **给 IP 方建议 + 理由** → **待裁决单选题** |
| **IP 已锁边界** | 见 §6 — Agent **不得** 在审议中重新打开 |
| **后台执行** | **默认禁止连案全自动** · 仅 **IP 逐关 LOCK** 后可开后台（§7） |

## 1. 触发条件（何时走本工作流）

走本流程，当且仅当议题属于以下之一：

1. **产品形态** — 页数、版式、道具形态、语音/实体边界  
2. **年龄/篇幅/阅读阻力** — 与 [`篇幅与单位构架_V1.1.md`](./篇幅与单位构架_V1.1.md) 冲突或需修订  
3. **Vol 结构 / 角色出场** — 与 F1、年级表、观察社引擎冲突  
4. **跨域 trade-off** — 文字 vs 图 vs 心理 vs 出版成本  
5. **伦理/校园真实** — 采访核实、误导尺度、日本在场  
6. IP 方明确说：**「专家组审议一下」「开个议题表」**

**不走本流程**（走常规 Agent / Skill 即可）：

- 已定正典内的执行（写稿、lint、日译、插图 brief）— **仍须 IP 逐关 LOCK 后才可后台**（§7）  
- 单一域、无争议的微调（如改一处读音标注）  
- 用户已给出 **单行 LOCK** 指令（直接执行并记入决策记录）

## 2. Agent 第一步 · 向 IP 方呈现议题

用户抛出开放问题后，Agent **必须先** 输出 **议题呈现块**（短，可扫读）：

```markdown
## 待审议议题（请确认或补充）

**议题一句话**：……

**背景（≤3 行）**：……

**可选方案**（若有）：
| 选项 | 摘要 | 主要代价 |
|------|------|----------|
| A | … | … |
| B | … | … |

**您要先拍板的前置问题**（若有）：……
```

规则：

- 有明确 A/B/C 时必须列表；无选项则列 **「待澄清维度」**  
- **禁止** 在第一步就写「专家组认为应该……」长结论  
- IP 方补充后，进入 §3 域分类与模拟审议

---

## 3. 域分类 · E01–E25 映射速查

Agent 将议题 **拆成 1–5 个子议题**，每子议题标注 **主责 E 编号 + 辅责**：

| 域标签 | 典型 E 编号 | 判断什么 |
|--------|-------------|----------|
| 文字/日文 | E04 · E07 | 文体、读音标注、校园语感、标日对齐 |
| 美工/视觉 | E06 · E22 | 插图档、面积比、角色识别、印前 |
| 心理/年龄 | E02 · E03 | 10–12 岁尺度、注意力、阅读阻力 |
| 阅读/结构 | E03 · E05 | 章节、桥梁书节奏、公平线索 |
| 校园/文化 | E07 · E09 | 名古屋小学、部活动、采访伦理 |
| 科学/本格 | E08 · E13 | 原理可验证、实验页、勿教辅化 |
| 出版/商业 | E16 · E20 · E22 | 页数、成本、竞品、出厂标准 |
| 伦理/钩子 | E10 · E02 | 好奇钩子、误导温柔度、禁对立社团 |
| IP 总控 | **E01** | 系列化、人物量、什么 **不该加** |

## 4. 模拟审议 · 三层流程

### 4.1 专家小组（单域）

- **组成**：该子议题 **主责 E** 1 类 + 必要时 1 辅责  
- **产出**：立场（✅采纳 / ⚠️附条件 / ❌否决）+ **≤5 句理由** + **对 IP 边界是否触碰**

### 4.2 跨学科专题组（多域）

- **触发**：子议题 ≥2 个 E 域，或存在 **文字↔图↔心理** 张力  
- **组成**：相关 E 代表 **圆桌**（文档内以 E04+E06+E03 等标注）  
- **产出**：**一致项** · **分歧项** · **折中方案**（若有）

### 4.3 大专家团交叉审议（E01 主持）

- **E01** 汇总各组，做 **交叉争议表**  
- **裁决规则**：  
  1. 触碰 §6 IP 边界 → **直接否决**，不进入待选  
  2. P0 正典（门禁、年级、Vol1 方案 B）→ **不可静默覆盖**  
  3. 平票 → **降 scope** 优先于 **加功能**（E01 原则：判断什么不该加）  
  4. 仍平票 → 写入 **待裁决单选题** 交给 IP 方

## 6. IP 边界锁（审议中不可重新打开）

以下在 **任何专家组模拟** 中视为 **已 LOCK**；若 IP 方要改，须 **显式解锁** 并另开议题：

| 边界 | 定案 |
|------|------|
| **语音包** | **无** — 无朗读音频 · 无 QR 配音 · 无 TTS 附件 |
| **实体道具包** | **无** — 无印刷可撕线索卡实物 · 无实验材料盒 · 无线圈本赠品 |
| **数字道具** | **仅** 手绘电子 PNG 草图 · **8 张** 最小包 · 嵌入 PDF/EPUB |
| **装帧** | **无** 线圈装帧设计 · 胶装/无线胶桥梁书 |
| Vol1 正典 | 方案 **B** · 《觉得奇怪，就先观察》· A001–A005 |
| 首市场 | **日本** · 主读者 **10–12 岁** |

## 7. 后台执行约定

### 7.0 总原则（IP 锁 · 2026-06-05）

| 项 | 定案 |
|----|------|
| **默认** | **禁止** 连案全自动 · **禁止** 未 LOCK 即在后台改正典/正文 |
| **允许后台** | 仅当 IP 方对 **某一关** 给出 **显式 LOCK** 之后 |
| **前台** | 议题呈现 · 专家组审议 · 待裁决单选题 · **每关验收摘要** |

**一句话**：先审议、再 LOCK、再后台；**一案一关**，过关再下一关。

---

### 7.1 什么算「一关」LOCK

IP 方须用 **可识别的 LOCK 句**（示例）：

```text
A004 CN 扩写 — LOCK · 开后台
案02 分镜 — LOCK · 只改样章包
Vol1 瑆层规则 — 专家组审议 · 暂不执行
```

| LOCK 粒度 | 示例 | 后台可做什么 |
|-----------|------|--------------|
| **单案 · 单工序** | 「A003 JP 日译 LOCK」 | 仅该案该工序文件 + 跟踪表一行 |
| **文档/sync** | 「README 同步 01_正文 LOCK」 | 仅列出的 md/索引 |
| **审议结论** | 「Q1=A LOCK · 写版式定案」 | 仅 §5 建议中已 LOCK 条目 |

**不算 LOCK**（Agent **不得** 开后台执行）：

- 「自动继续」「按进度计划完成」「后台跟进 Vol1」等 **无边界** 指令  
- 专家组 **建议** 但未勾选待裁决选项  
- 推断 IP 方「应该会同意」

---

### 7.2 默认禁止（连案全自动）

以下 **一律禁止**，除非 IP 方 **另开议题** 并 **显式解锁** 本节约束：

| 禁止项 | 说明 |
|--------|------|
| **连案正文** | 一次后台扩写 / 日译 **≥2 案** |
| **连案 + 分镜 + 图** | 文字与视觉 **捆批** 无人审 |
| **静默改正典** | 审议未 LOCK 即写入 LOCKED / ACTIVE 正典 |
| **整卷 PRODUCT 冲刺** | 「五案齐 · E07 · PDF」类 **端到端** 无人关 |

Vol1 生产默认节奏 → **§7.3 关卡链**（逐关 LOCK，不跳关）。

---

### 7.3 Vol1 推荐关卡链（一案一关）

每关后台完成后，**只** 在单独通知或跟踪表报：**交付路径 + 计数/门禁 + 下一关待 LOCK 项** — **不** 在主线聊天刷 diff。

```
案 N：CN 结构/扩写 → 【IP 审】→ JP → 【IP 审】→ 分镜 → 【IP 确认】→ 下一案
卷级：五案文字齐 → 序/瑆 → 【IP 审】→ 插图发包 → E07 → PDF
```

| 场景 | 前台（对话 UI） | 后台（须 §7.1 LOCK） |
|------|-----------------|---------------------|
| 开放式议题 | 议题呈现 + 专家组审议 + 单选题 | **不启动** |
| IP **已 LOCK** 的单关任务 | 一行 ACK + 指向路径 | 仅 LOCK 范围内写文件 · lint · 索引 · 跟踪表 |
| 单文件微调（无争议） | 可前台直接改 | 可选后台，但 **仍须** IP 一句 LOCK |
| 重复性 lint | 失败时 **首条错误 + 文件** | `scripts/pre_push_check.py`（不改故事） |

原则：**已裁决的不在聊天里重演辩论**；**未 LOCK 的不在后台动正文/正典**；**Never 连案无人审**。

## 9. 快速示例（缩略）

**IP 问**：「Vol1 要不要加语音朗读？」

1. **议题呈现**：选项 A 无 / B 样章 QR / C 全卷音频 — 注 §6 已锁无语音  
2. **域分类**：E03 阅读 · E16 出版 · E20 商业  
3. **小组**：E16 ❌ 成本与年龄注意力 · E03 ⚠️ 日读 15–20 分钟已够  
4. **大团**：E01 裁决 — **维持无语音**；若 IP 坚持，须 **显式解锁 §6** 并单独立项  
5. **建议**：不做语音；薄样张 PDF 试读即可  
6. **待裁决**：无（已锁）或「是否显式解锁语音边界？」单选题
