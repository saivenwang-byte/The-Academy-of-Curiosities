# 《学堂趣事录》IP 工作室 · 出版协同组织模型 V1.0

> **Status**: ACTIVE · **组织正典**（范式层）  
> **模式**: **IP 工作室主导 + 出版社协同 + 专家池审校 + Agent/Skill 日常质控**  
> **原则**: 专家不替代创作 · Agent 不替代判断 · 出版社不替代 IP 方向 · 市场不牺牲儿童适配  
> **关联**: [`12_人类专家池`](./12_人类专家池_V1.0.md) · [`11_专家池与Agent调度`](./11_专家池与Agent调度_v0.1.md) · [`14_专家蒸馏执行进度`](./14_专家蒸馏执行进度.md)  
> **阶段状态**: [`15_出版全流程阶段状态.md`](./15_出版全流程阶段状态.md)  
> **最后更新**: 2026-06-04

## 0. 为什么用这个模式

《学堂趣事录》不是单本书，是 **可延展儿童 IP**。  
比「作者写完交给出版社」更适合：

| 方 | 掌握什么 |
|----|----------|
| **IP 方** | 世界观 · 人物 · 长期路线 |
| **出版社** | 图书产品化 · 编辑出版 · 印刷发行 |
| **专家池** | 关键节点校准（建议/校验/否决边界） |
| **Agent/Skill** | 日常检查 · 标准化质控 |

## 二、IP 委员会（大脑）

### 组成与权责

| 角色 | 权责 | 本项目映射 |
|------|------|------------|
| **IP 出品人 / 项目负责人** | 预算 · 合作 · 最终 IP 方向 | 主编 |
| **IP 总策划 / Showrunner** | 世界观 · 人物 · 系列路线 · **什么不加** | `series-architect` · 11_ §5 总控模板 |
| **系列主编** | 文本质量 · 读者定位 · 书系结构 | 卷任务包 · B2 节奏 |
| **美术总监** | 画风 · 角色 · 视觉一致 | `visual-auditor` · V12–14 |
| **出版负责人** | 可行性 · 成本 · 周期 · 渠道 | E16 · 待出版社 |
| **儿童内容安全负责人** | 伦理边界 · **否决权** | S18–S20 · E13 · 红线 Rule |

### 委员会只决大问题

- 人物是否新增 · 山本理紗何时登场 · 中谷琦尺度  
- Vol1 是否只做入社线 · 画风 8–10 岁 · 是否进印厂 · 是否授权延展  

> **专家可建议，决策必须集中** — 否则项目越散。

## 十五、四条权责铁律（结论）

1. **IP 方控方向** — 出版社改包装，不改观察社内核。  
2. **作者写故事，编辑控结构，专家控边界** — 权责分离。  
3. **美术控视觉，出版控产品，印刷控制作** — 阶段门禁。  
4. **试读/市场反馈反哺下一卷** — 阶段 14 闭环。
